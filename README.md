# 电池统计

电池统计是一款面向 Root 用户的 Android 功耗观察工具。它把长期采样放在 Root 模块里执行，前台应用主要负责读取、展示和临时请求实时数据，尽量减少应用本身常驻带来的额外耗电。

## 主要功能

- 总览：显示内存、GPU、CPU 核心频率/占用、电池功率、电压、温度和守护状态。
- 耗电统计：记录放电过程、平均功耗、已使用时长、亮屏时长、还能使用和还能亮屏使用。
- 应用耗电：区分前台与后台耗电，支持隐藏/显示系统应用，并按时长或功耗排序。
- 充电统计：记录充电功率、电量、电压、温度和充电曲线。
- 进程信息：读取进程 CPU、RES 内存占用，支持搜索、排序和应用图标显示。
- 悬浮小窗：可在其他应用上方实时查看单个进程的线程 CPU 占用、运行核心、TID 和线程名。
- 智能实时采样：进入总览/耗电/充电页面且应用在前台、屏幕点亮时，模块会临时启用 OPPO/realme 电量计 1 秒刷新；退后台、熄屏、切到进程页或应用被杀后自动恢复低频。
- Root 守护：模块负责低频采样，支持模块管理器动作按钮或应用内一键启停。
- 深色模式：跟随系统自动切换。

## 当前版本

`3.0.0`

## 安装方式

1. 安装签名 APK：`battery-stats-v3.0.0-release-signed.apk`
2. 在 Magisk、APatch 或 KernelSU 管理器中刷入模块：`battery-stats-v3.0.0-magisk-module.zip`
3. 重启，或在管理器中执行模块动作按钮启动采样。
4. 打开应用并授予 Root 权限。

## 数据与采样

模块默认把采样数据写入：

- `/data/adb/battery_stats/samples.csv`
- `/data/adb/battery_stats/app_usage.csv`
- `/data/adb/battery_stats/process_top.csv`

长期后台统计仍然是低频策略；只有前台查看实时电池页面时才会请求高频电量计刷新。高频请求由模块兜底管理，即使应用被杀，模块也会在检测到应用不在前台后关闭高频。

## 构建

```powershell
.\build_apk.ps1
.\build_magisk_module.ps1
```

签名配置通过本地 `signing.properties` 和 `release.keystore` 提供，这两个文件不会提交到仓库。
